News worker — 历史评估

📖 这个页面是干什么用的

News worker 每 15 分钟从 CoinDesk RSS 抓最新加密新闻，让 LLM 综合判断 未来 1-4h 短线影响， 结果（high / medium / low / none）会被注入 scorer 的 prompt，影响 TradingView 信号的打分。

本页的用途：人肉验证 LLM 的判断是否合理 —— 防止它把炒作误标 high、把真黑天鹅误标 low。 具体怎么用：

  1. 每天扫一眼列表，看 impact 分布是否合理（normal 时段应大多 low/none，事件时段才出 high）
  2. 对每条 HIGH 点进详情，看 LLM 给的 reasoning + 每条新闻的独立判断是否扛得住质询
  3. 发现误判 → 收集 case → 改 internal/agent/news/classifier.go 里的 prompt → 灰度
  4. 跑 1-2 周累积足够样本后，才考虑把"新闻 → 主动下单/平仓"放到下一个 spec

数据库表 news_snapshots；想直接 SQL 查也行。
